Overview

This project builds a leakage-safe, year-conditional donor lapse prediction pipeline using opportunity-level donation records from a nonprofit Salesforce CRM. The goal is to rank donors by lapse risk and enable targeted, resource-constrained outreach. The main implementation is provided in a single Jupyter Notebook that performs the full pipeline end-to-end: data loading, cleaning, donor-year panel construction, leakage-safe labeling, model training, evaluation, calibration, and Top-K outreach analysis.

Data Requirements

The notebook expects a CSV file containing opportunity-level donation records. By default it looks for:
RAW_PATH=/workspace/opportunity_enrichedshared.csv
CLEAN_PATH=/workspace/clean_opportunities.csv
Required columns: account_id, opportunity_id, close_date, amount

Environment Requirements

Python 3.10+
Required packages: numpy, pandas, matplotlib, scikit-learn, xgboost, lightgbm, tabulate
Optional: torch (GPU availability check only)
The notebook includes a bootstrap cell that automatically installs missing packages. To install manually:
pip install numpy pandas matplotlib scikit-learn xgboost lightgbm tabulate jupyter

How to Run

Option A - Simplest (RunPod/workspace):
Place the raw dataset at /workspace/opportunity_enrichedshared.csv
Follow RunPodSetupInstructions.txt to set up your environment
Open the notebook and run all cells from top to bottom
Cell 0 is optional if your environment is already stable
Cells 4-6 (cleaning) can be skipped if the cleaned dataset is already available

Option B - Custom file paths (local machine):
export RAW_PATH="/absolute/path/to/opportunity_enrichedshared.csv"
export CLEAN_PATH="/absolute/path/to/clean_opportunities.csv"
export SAVE_CLEAN="1"
export RUN_ENV_NOTE="Local environment"
jupyter notebook

Pipeline Summary

Section A: Data cleaning - removes PII, normalizes types, deduplicates transactions
Section B: EDA - validates coverage and temporal donation trends
Section C: Donor x Year panel construction, leakage-safe feature engineering, and year-conditional lapse labeling
Section D: Temporal train/val/test split (2014-2023 / 2024 / 2025) and model training (Logistic Regression, Random Forest, XGBoost)
Section E: Probability calibration (Platt scaling) and Top-K outreach policy evaluation

Reproducibility Notes

A single random seed (RANDOM_SEED = 42) is used throughout.
File paths are controlled via environment variables - no notebook edits required.
The cleaned dataset can be regenerated by setting SAVE_CLEAN="1".
The train/val/test split is temporal - run cells in order to preserve split logic.

Key Results (Test Year: 2025)

XGBoost AUPRC: ~0.75
Precision@1000: ~79%
Lift@1000: ~1.26x over random outreach
